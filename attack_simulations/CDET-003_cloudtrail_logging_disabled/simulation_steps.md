# CDET-003 — Simulation Steps: CloudTrail Logging Disabled

## IMPORTANT WARNINGS

> **WARNING**: Stopping CloudTrail logging in a production or shared account will create an audit gap and may violate compliance requirements (PCI-DSS, SOC2, HIPAA, FedRAMP). Only execute this simulation in an **isolated test account** with explicit written authorization.
>
> **WARNING**: `DeleteTrail` is irreversible. Do NOT execute it unless you are in a completely isolated sandbox account and have a trail configuration backup (Terraform/CloudFormation).
>
> **RECOMMENDATION**: Execute `StopLogging` and `UpdateTrail` variants only — skip `DeleteTrail` entirely unless your test environment is fully disposable.

---

## Prerequisites

### Required AWS Permissions
- `cloudtrail:DescribeTrails` — enumerate existing trails
- `cloudtrail:GetTrailStatus` — check logging status
- `cloudtrail:StopLogging` — Variant 1
- `cloudtrail:UpdateTrail` — Variant 2
- `cloudtrail:StartLogging` — for cleanup after Variant 1
- `cloudtrail:DeleteTrail` — **ONLY for isolated sandbox accounts** (Variant 3)

### Pre-flight Check
```bash
# Verify identity
aws sts get-caller-identity

# List all CloudTrail trails in this account
aws cloudtrail describe-trails \
  --include-shadow-trails false \
  --query 'trailList[].{Name:Name, HomeRegion:HomeRegion, MultiRegion:IsMultiRegionTrail, S3Bucket:S3BucketName, LoggingEnabled:HasCustomEventSelectors}' \
  --output table

# Check current logging status of each trail
export TRAIL_NAME="management-events"  # Replace with your trail name
aws cloudtrail get-trail-status \
  --name "$TRAIL_NAME" \
  --query '{IsLogging:IsLogging, LatestDelivery:LatestDeliveryTime}'
```

## Variant 1: StopLogging (Primary Simulation)

**Risk level**: High (creates audit gap) | **Reversible**: Yes

```bash
export TRAIL_NAME="management-events"  # Replace with actual trail name
export AWS_REGION="us-east-1"

# Step 1: Confirm current status (it should be logging)
aws cloudtrail get-trail-status \
  --name "$TRAIL_NAME" \
  --query 'IsLogging'
# Expected: true

# Step 2: Stop logging
# WARNING: This immediately stops CloudTrail event recording for this trail.
aws cloudtrail stop-logging \
  --name "$TRAIL_NAME"

# Expected output: (no output on success — exit code 0)

# Step 3: Confirm logging is stopped
aws cloudtrail get-trail-status \
  --name "$TRAIL_NAME" \
  --query '{IsLogging:IsLogging, LatestDelivery:LatestDeliveryTime}'
# Expected: {"IsLogging": false, ...}
```

**CloudTrail event generated**: `StopLogging`

### Variant 1 Cleanup — Restart Logging

```bash
# IMPORTANT: Restart logging immediately after simulation
# WARNING: This restarts logging — all events going forward will be captured
aws cloudtrail start-logging \
  --name "$TRAIL_NAME"

# Confirm logging is restored
aws cloudtrail get-trail-status \
  --name "$TRAIL_NAME" \
  --query 'IsLogging'
# Expected: true

echo "Logging restored for trail: $TRAIL_NAME"
```

**CloudTrail event generated by cleanup**: `StartLogging`

## Variant 2: UpdateTrail Degradation (Stealthiest)

**Risk level**: Medium (degrades visibility) | **Reversible**: Yes

This variant disables management event logging and global service events without triggering `StopLogging`. Many SIEM rules alert on `StopLogging` but not on `UpdateTrail` parameter changes.

```bash
# Step 1: Record current trail configuration for restoration
aws cloudtrail get-event-selectors \
  --trail-name "$TRAIL_NAME" \
  --output json > /tmp/original_event_selectors.json

echo "Original event selectors saved to /tmp/original_event_selectors.json"

# Step 2: Get current trail settings
aws cloudtrail describe-trails \
  --trail-name-list "$TRAIL_NAME" \
  --output json > /tmp/original_trail_config.json

# Step 3: Degrade the trail — disable management events
# WARNING: This removes IAM, STS, KMS, CloudTrail events from the trail.
aws cloudtrail put-event-selectors \
  --trail-name "$TRAIL_NAME" \
  --event-selectors '[{"ReadWriteType": "All", "IncludeManagementEvents": false, "DataResources": []}]'

# Expected output: (updated event selectors JSON)

# Step 4: Also disable global service events (IAM actions)
# WARNING: This means IAM actions (CreateUser, AttachPolicy etc.) will no longer be logged.
aws cloudtrail update-trail \
  --name "$TRAIL_NAME" \
  --no-include-global-service-events

# Expected output: Trail configuration JSON with IsLogging: true but degraded
```

**CloudTrail events generated**: `PutEventSelectors`, `UpdateTrail`

### Variant 2 Cleanup — Restore Full Logging

```bash
# Restore management events
aws cloudtrail put-event-selectors \
  --trail-name "$TRAIL_NAME" \
  --event-selectors '[{"ReadWriteType": "All", "IncludeManagementEvents": true, "DataResources": []}]'

# Restore global service events
aws cloudtrail update-trail \
  --name "$TRAIL_NAME" \
  --include-global-service-events

echo "Trail configuration fully restored."

# Verify restoration
aws cloudtrail get-event-selectors --trail-name "$TRAIL_NAME"
```

## Variant 3: DeleteTrail (**SANDBOX ONLY**)

> **DANGER**: Only execute in a fully isolated AWS account that will be discarded after testing.  
> **DO NOT** execute in any account with production workloads, shared infrastructure, or compliance requirements.

```bash
# SANDBOX ONLY — verify you are in the correct account before proceeding
aws sts get-caller-identity
# Confirm AccountId matches your disposable sandbox account

# WARNING: The following command permanently deletes the CloudTrail trail.
# There is no undo. You must recreate it from scratch.
aws cloudtrail delete-trail \
  --name "$TRAIL_NAME"

# Expected output: (no output on success — exit code 0)
```

**CloudTrail event generated**: `DeleteTrail`  
**Note**: This is the *last* event that will appear for this trail. After deletion, no further events are captured by this trail.

### Variant 3 Recovery — Recreate Trail

```bash
# Recreate the trail (adjust parameters to match original configuration)
aws cloudtrail create-trail \
  --name "$TRAIL_NAME" \
  --s3-bucket-name "your-cloudtrail-bucket" \
  --include-global-service-events \
  --is-multi-region-trail \
  --enable-log-file-validation

aws cloudtrail start-logging --name "$TRAIL_NAME"
```

